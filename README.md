Mail-Order-Pharmacy

Mail Order Pharmacy Project made for the completion of GenC Next Internship from Cognizant.

Problem Statement - A leading pharma organization wants to develop a mail-order pharmacy using miroservices. The organization is a middle man between an Insurance Company and its members. But, because of its size, it can negotiate large customer contracts and negtiate with both pharmacies and pharma companies to get best prices for drugs. Thus, Mail Order Pharmacy is a solution which operates through your health insurance plan and can be cheaper and more convenient than getting medicines from a local pharmacy. It gives a sense of convenience where the medicines are delivered to users' doorstep.

Made using Microservices Architecture with the follwoung microservices - 
1. Member Portal - A view for the consumers
2. Authorization - For authentication purposes
3. Drugs - To manage availability and details of drugs and their sellers
4. Subscription - To manage subscriptions to a particular drug
5. Refill - To track when a refill would be required for a user

Technologies Used-
Frontend - JSP
Backend - Spring Boot (Spring MVC, Spring Data JPA), H2 in Memory Database
Testing - JUnit, Mockito
